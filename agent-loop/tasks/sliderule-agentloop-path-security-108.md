# SlideRule AgentLoop 108: path security

## Execution status
- Status: pending
- Goal: enforce safe filesystem boundaries for Python AgentLoop run and artifact readers.
- Required gate: `slideruleAgentLoopPathSecurity108Gates`

## Context
The Python control plane will read files generated by Node AgentLoop. This must be guarded before exposing artifact and log APIs.

## Allowed files
- `slide-rule-python/services/agent_loop_paths.py`
- `slide-rule-python/services/agent_loop_runs.py`
- `slide-rule-python/tests/test_agent_loop_path_security.py`
- `agent-loop/tasks/sliderule-agentloop-path-security-108.md`
- This task file

## Do not
- Do not allow absolute user-supplied paths.
- Do not follow symlink escapes outside the allowed root.
- Do not expose raw filesystem errors to API callers.

## Acceptance criteria
- Add a test named `agentloop path security 108 rejects traversal and absolute escapes`.
- Allowed roots include only documented run and artifact roots.
- Traversal, absolute path, symlink escape, and drive-prefix escape cases are rejected.
- Readers use the path helper instead of open-coding path joins.
