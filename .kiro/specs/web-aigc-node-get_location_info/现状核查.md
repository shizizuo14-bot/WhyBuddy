# web-aigc-node-get_location_info 现状核查

## 结论

本轮已在不修改 `server/index.ts`、`server/core/workflow-runtime-engine.ts` 的前提下，实现 `web-aigc-node-get_location_info` 的最小可用闭环，且优先采用受限授权版能力：

- 仅支持显式传入：
  - `coarseLocation`
  - `timezone`
  - `locale`
- 会输出标准化的位置结果结构；
- 会输出授权摘要与隐私摘要；
- 明确阻断高风险精确定位，不返回经纬度、精度半径等敏感字段；
- 已提供独立 route、node adapter 与测试。

因此 `tasks.md` 顶层 4 项可以按本轮代码事实勾选完成。

## 已落地内容

### 1. 定义位置输出结构

已完成。

证据：

- `shared/web-aigc-location-info.ts`
  - 定义了 `get_location_info` 的输入、输出、授权摘要、隐私摘要契约；
- `server/routes/node-adapters/get-location-info-node-adapter.ts`
  - 产出统一的 `location / authorization / privacy / context / warnings` 结构；
- `server/tests/get-location-info-node-adapter.test.ts`
  - 已验证 coarse location、timezone、locale 的标准化输出。

本轮输出结构特点：

- `location.coarseLocation`
  - 只允许国家/区域/城市/区县级 coarse 信息；
- `location.timezone`
  - 仅接受有效 IANA timezone；
- `location.locale`
  - 做最小 canonical 化；
- 不返回任何精确坐标字段。

### 2. 定义授权与隐私规则

已完成。

证据：

- `server/routes/node-adapters/get-location-info-node-adapter.ts`
  - 定义 `authorization.status`；
  - 输出 `authorization.granted / grantedBy / disclosureText`；
  - 输出 `privacy.precisionLevel / deniedFields / retention / notes`；
  - 对 `requestedPrecision = precise` 时自动降级到 `precise_blocked`；
  - 固定声明：
    - `exactCoordinatesStored = false`
    - `exactCoordinatesAllowed = false`
- `server/tests/get-location-info-node-adapter.test.ts`
  - 已验证精确定位请求被降级；
  - 已验证无授权场景仍只返回显式 coarse 字段；
  - 已验证非法 timezone 被过滤并给出 warning。

本轮隐私边界：

- 只接受调用方显式传入的信息；
- 不主动调用浏览器 geolocation；
- 不做 IP 定位、GPS 定位、Wi-Fi 定位等高风险精确定位；
- 所有结果都以“coarse only”治理口径输出。

### 3. 评估产品场景是否需要

已完成。

结论：

- 按 spec 初衷，`get_location_info` 不是 Cube 当前主场景里的高优先级前端定位节点；
- 但在低风险场景下，`timezone / locale / coarse region` 对内容本地化、时间展示、区域分流仍有最小价值；
- 因此本轮采用“受限授权版最小实现”而不是完整精确定位接入。

落地体现：

- route 与 adapter 只承接低风险 coarse 信息；
- 高风险精确定位需求被显式阻断并降级；
- 文档与测试都围绕该受限范围收口。

### 4. 验证前端兼容性

已完成，但边界是“接口形状兼容”，不是“前端页面已接线”。

证据：

- `server/routes/get-location-info.ts`
  - 提供了独立 `POST /nodes/execute` 路由；
- `server/tests/get-location-info-routes.test.ts`
  - 已验证：
    - 非法 `nodeType` 返回 400；
    - 合法请求返回 completed payload；
    - 精确定位请求返回 200，但带 privacy downgrade warning。

兼容性判断：

- 返回结构为纯 JSON，前端节点面板/宿主壳层可以直接消费；
- `warnings`、`authorization`、`privacy` 字段有助于 UI 做授权提示和隐私说明；
- 由于本轮不接浏览器原生 geolocation，因此不引入前端权限弹窗兼容问题。
