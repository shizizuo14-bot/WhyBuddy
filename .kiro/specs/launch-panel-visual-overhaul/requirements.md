# 需求文档：发起面板视觉大改

## 目标

基于效果图，将当前的任务发起浮层面板从基础布局升级为完整的"任务自动驾驶"发起体验。效果图展示了一个深色遮罩上的居中浮层卡片，包含模式切换、目标输入、自主规划路线流程图、能力驾驶舱网格和输出交付配置。

## 需求

### 需求 1：浮层卡片与遮罩

**用户故事：** 作为用户，我希望发起面板以居中浮层形式出现在深色半透明遮罩上，视觉焦点集中在发起内容上。

#### 验收标准

1. 系统应在打开发起面板时在 3D 场景上方显示深色半透明遮罩（约 `rgba(0,0,0,0.5)`），背后的 3D 办公室场景应保持可见
2. 系统应在遮罩上居中显示一个白色圆角卡片（最大宽度约 760px，最大高度 `calc(100vh - 120px)`，内部可滚动）
3. 面板必须使用 `position: fixed; inset: 0; z-index` 覆盖在 3D 场景之上，不能脱离视口或撑出页面
4. 卡片应有标题栏：左侧"任务自动驾驶"标题 + "Autopilot Control"副标题，右侧"更新任务"、"刷新"、"更多"操作按钮
5. 点击遮罩或按 Esc 应关闭面板

### 需求 2：模式切换 Tab

**用户故事：** 作为用户，我希望在发起任务时选择不同的执行模式，以匹配任务复杂度。

#### 验收标准

1. 系统应在标题栏下方显示横向 Tab 切换栏，包含：快速模式、标准模式、深度模式、研究模式、自定义模式
2. 默认选中"标准模式"，选中态使用深色填充 + 白色文字
3. 切换模式时应更新下方能力驾驶舱的可用能力展示

### 需求 3：目标输入区

**用户故事：** 作为用户，我希望在一个清晰的文本框中输入任务目标、约束和交付要求。

#### 验收标准

1. 系统应显示"输入你的目标"标题和提示文字
2. 系统应提供多行文本输入框，带字数统计（如 0/1000）
3. 提示文字应说明可输入的内容类型：目标、约束、交付物、截止时间等

### 需求 4：自主规划路线流程图

**用户故事：** 作为用户，我希望看到系统将如何自主规划和执行我的任务的可视化流程。

#### 验收标准

1. 系统应显示"自主规划路线"区块标题和说明文字
2. 系统应显示四步横向流程图：目的地（理解目标与约束）→ 路线搜索（分解任务与策略）→ 执行步骤（智能分配与执行）→ 校验/证据（证据链与验证记录）
3. 每个步骤应有图标、标题和简短描述
4. 步骤之间用箭头或连接线连接

### 需求 5：能力驾驶舱 COCKPIT

**用户故事：** 作为用户，我希望看到系统可以调用哪些能力来完成我的任务。

#### 验收标准

1. 系统应显示"能力驾驶舱 COCKPIT"区块标题和说明文字
2. 系统应以网格形式展示可用能力卡片：浏览器能力（真实浏览器/操作）、代码执行器（运行 Python 代码）、文件系统（读写与管理文件）、知识检索（语义搜索/问答）
3. 应有"更多能力"展开按钮
4. 每个能力卡片应有图标和简短描述

### 需求 6：输出与交付配置

**用户故事：** 作为用户，我希望预先配置任务完成后的输出形式。

#### 验收标准

1. 系统应显示"输出与交付"区块标题和说明文字
2. 系统应以 chip 形式展示可选交付物：结果摘要、生成文件、执行日志、详细截图、操作记录
3. 应有"完成后自动打开结果"开关

### 需求 7：底部操作栏

**用户故事：** 作为用户，我希望在面板底部有清晰的操作入口来附加文件、调整设置和启动任务。

#### 验收标准

1. 系统应在面板底部显示操作栏，包含：添加附件按钮、高级设置按钮、保存为模板按钮、启动任务主按钮
2. "启动任务"应为深色主按钮，带播放图标
3. 操作栏应与上方内容区有分隔线
