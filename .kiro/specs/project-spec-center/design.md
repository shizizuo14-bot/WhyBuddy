# 设计文档：Project Spec Center

## 设计概述

Spec Center 是 Project-first 系统的稳定骨架。没有 spec，自动驾驶容易基于模糊目标直接执行；有 spec，路线规划、执行验证和证据回流都有参照。

## Spec 结构

第一阶段建议使用 Markdown 模板：

```md
# {Project Name} Spec v{version}

## 1. 项目目标

## 2. 用户与场景

## 3. 功能范围

## 4. 非目标

## 5. 技术约束

## 6. 数据与权限

## 7. 交付物

## 8. 验收标准

## 9. 风险与假设

## 10. 来源与决策
```

## Spec 生命周期

```text
draft -> reviewing -> accepted -> superseded
```

### draft

系统根据项目目标、问答和澄清生成草案。

### reviewing

等待用户确认或修改。

### accepted

可作为路线规划和执行依据。

### superseded

被新版本替代。

## 完整度维度

```ts
interface SpecCompleteness {
  goal: number;
  audience: number;
  scope: number;
  constraints: number;
  deliverables: number;
  acceptance: number;
  risks: number;
}
```

完整度不应成为复杂评分秀，而应用于判断下一步：

- 继续澄清
- 生成 spec
- 进入路线规划
- 允许深度执行

## 来源追踪

每个 spec 版本记录：

- source messages
- source evidence
- source artifacts
- decisions
- generated by

## 与 Route Planner 的关系

Route Planner 输入：

- current spec
- spec completeness
- risks
- project constraints
- recent decisions

Route Planner 输出回写：

- route 引用 specId
- route risk 引用 spec risk
- route steps 引用 spec deliverables

## 非目标

- 第一阶段不做复杂协同编辑。
- 第一阶段不做完整需求管理系统。
- 第一阶段不做多人审批。
