# 需求文档：Project Execution Center

## 目标

将 `/tasks` 从全局任务中心升级为项目执行中心。任务中心不再承担主发起入口，而是负责项目下的执行队列、运行详情、人工接管、失败恢复、证据查看和回放。

## 需求

### 需求 1：项目过滤

任务中心应默认展示当前项目相关任务，并支持切换项目或查看未归档任务。

### 需求 2：任务归属清晰

每个任务应显示所属项目、关联路线、关联 spec 和来源输入。

### 需求 3：执行队列视图

任务中心应展示项目下 queued、running、waiting、completed、failed、cancelled 等任务状态。

### 需求 4：接管动作项目化

终止、重试、继续、人工决策、路线调整等 operator action 应写入项目 evidence 和 message。

### 需求 5：证据和日志归档

任务运行日志、agent 状态、runtime 事件、产物链接应归档到当前项目。

### 需求 6：不抢首页入口

任务中心不应突出“新建任务”作为主入口；需要新建时应引导用户回到当前项目驾驶舱继续推进。

### 需求 7：支持未绑定历史任务

历史任务没有 `projectId` 时，应展示为未归档任务，并支持手动归入某个项目。

### 需求 8：路线与角色可解释

任务详情应能说明任务来自哪条路线、由哪些 FSD 角色执行、调用了哪些角色内部能力类别，但不把 50+ 节点暴露成用户主入口。

### 需求 9：失败恢复

失败任务应提供失败原因、可恢复动作、建议路线调整和重试风险，并写回项目。

### 需求 10：兼容现有任务中心

改造应复用现有 `/tasks` 页面、任务 store、operator action、dialog 和测试基础，不应破坏当前任务操作能力。
