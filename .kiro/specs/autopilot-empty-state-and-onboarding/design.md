# 设计文档：自动驾驶空状态与新手引导

## 设计概述

Onboarding 不是弹窗堆叠，而是把示例目的地和路线心智嵌入入口空状态。

## 引导结构

### 1. Empty State

- 一句话解释任务自动驾驶
- 三个示例目的地
- “为什么这样输入”说明

### 2. Example Routes

示例：

- 快速：直接创建 mission
- 标准：规划路线 + 编队 + 证据
- 深度：附件/团队/多阶段
- 补路标：目标不完整
- 高级执行：浏览器/命令/沙盒

### 3. Guided Tour

步骤：

1. 输入目的地
2. 查看路线
3. 选择路线
4. 观察执行
5. 接管关键点
6. 查看证据

## 回补既有缺陷方向

- 检查 README 示例与前端示例是否一致。
- 将 specs 中 L1-L5 边界转成用户能理解的提示。
- 避免前端示例承诺当前未实现的能力。
